# Part 5 / Clip 6: Workspace Builder로 내 에이전트 만들기

> 강의 영상: Part 5 / Clip 6 (~35분)
> 만드는 것: 본인 맞춤 에이전트 워크스페이스 1개

---

## 🎯 이 클립에서 만드는 것

드디어 오늘의 핵심입니다. 지금까지는 Claude Code 안에서 프롬프트를 잘 쓰고, 플러그인을 불러보고, 팀도 구성해봤죠. 이번에는 한 단계 더 갑니다. 내 업무를 계속 맡길 수 있는 **나만의 에이전트 워크스페이스**를 하나 만듭니다.

Workspace Builder는 이름 그대로 워크스페이스를 만들어주는 에이전트입니다. 직접 일을 처리하는 비서가 아니라, 비서를 만들기 위한 설계자예요. 여러분은 대화로 목적과 업무를 설명하고, Workspace Builder는 Phase 0부터 Phase 7까지 물어보면서 `90-output/{내에이전트}/` 아래에 새 워크스페이스를 생성합니다.

| Before | After |
|---|---|
| 매번 Claude Code에게 업무 설명을 처음부터 반복 | 내 업무용 CLAUDE.md와 커맨드가 있는 전용 워크스페이스 생성 |
| "보고서 써줘", "분석해줘"가 섞여 있음 | 아키타입과 커맨드가 정리된 비서로 분리 |
| 결과물이 대화창에만 남음 | `~/Desktop/Workspace_Builder/90-output/{내에이전트}/` 폴더로 남음 |
| 다음에 다시 쓰기 어려움 | 생성된 폴더에서 `claude` 실행 후 같은 방식으로 재사용 |

예시는 이렇게 잡으면 됩니다. `BPTC 자재관리 에이전트`, `물류 KPI 분석 에이전트`, `현장 회의록 정리 에이전트`, `사내 교육자료 작성 에이전트`. 멋진 이름보다 실제로 다시 부를 일이 있는 이름이 좋아요.

## 💡 핵심 개념

### Workspace Builder는 비서가 아니라 비서 제작소입니다

이 차이를 꼭 잡고 가야 합니다. Workspace Builder에게 "이번 달 자재 보고서 작성해줘"라고 하면 방향이 틀어집니다. 이 도구는 실제 업무를 처리하는 곳이 아니라, 그 업무를 처리할 전용 워크스페이스를 설계하고 생성하는 곳이에요.

Workspace Builder의 원칙은 단순합니다.

| 원칙 | 뜻 |
|---|---|
| One Workspace, One Agent | 하나의 워크스페이스는 하나의 특화 에이전트 역할을 맡음 |
| Archetype First | Writer / Study / Knowledge Base / Research / Development 중 하나에서 출발 |
| 대화 기반 설계 | 사용자가 설명하면 Claude가 이해한 내용을 확인하며 설계 |
| Phase 0 감사 | 새로 만들지, 기존 것을 개선할지, 검증할지 먼저 판단 |
| 다관점 검토 | 필요하면 Agent Council로 Codex와 Gemini 의견까지 받음 |

저는 여기서 "공장"이라고 생각하면 쉽다고 말합니다. Workspace Builder는 완제품을 바로 주는 직원이 아니라, 여러분 업무에 맞는 작업대와 도구함을 만들어주는 공장입니다. 만들어진 뒤에는 그 새 워크스페이스로 이동해서 실제 일을 시작해요.

### Phase 0부터 Phase 7까지 지나갑니다

이번 실습에서 경험할 흐름은 아래와 같습니다. 오늘 실제 생성은 Phase 6에서 끝나고, Phase 7은 나중에 에이전트를 계속 발전시키는 장치라고 보면 됩니다.

| Phase | 하는 일 | 학생이 해야 할 것 |
|---|---|---|
| Phase 0: 현황 감사 | 새 워크스페이스인지, 기존 개선인지 확인 | "새로 만들 겁니다"라고 분명히 답하기 |
| Phase 1: 컨텍스트 수집 | 목적, 사용자, 입력 자료, 결과물 파악 | 내 업무를 구체적으로 설명 |
| Phase 2: 아키타입 매칭 | 5가지 아키타입 중 추천 | Clip 5의 `my-archetype.md`와 비교 |
| Phase 3: 상세 설계 | 워크플로우, 커맨드, 지식 베이스 설계 | 커맨드와 폴더 구조 확인 |
| Phase 4: 설계 검토 | 전체 설계를 생성 전 최종 점검 | 빠진 것, 너무 큰 것, 위험한 것 보기 |
| Phase 5: 워크스페이스 생성 | 폴더, CLAUDE.md, 커맨드 파일 생성 | 생성 승인 |
| Phase 6: 완료 및 인수 | 생성 위치와 시작 방법 안내 | 생성된 폴더에서 실행 확인 |
| Phase 7: 진화 메커니즘 | 피드백과 검증 실패를 다음 개선으로 연결 | 다음 버전에서 고칠 질문 남기기 |

이 흐름은 조금 길어 보이지만, 실제로는 대화입니다. 질문에 답하고, 이해한 내용이 맞는지 확인하고, 마지막에 생성하면 됩니다.

## 📁 시작 전 — 자료 셋업

이번 클립은 Workspace Builder 폴더가 필요합니다. 먼저 강의 자료에서 `Workspace_Builder`를 받아 `~/Desktop/Workspace_Builder/`에 둡니다. 폴더 이름이 다르면 Claude가 경로를 못 찾는 학생이 꼭 나와요. 이름을 정확히 맞춰 주세요.

### 1단계. Workspace Builder 위치 확인

```bash
ls ~/Desktop/Workspace_Builder
```

`README.md`, `CLAUDE.md`, `QUICKSTART.md`, `10-standards/`, `20-archetypes/`, `90-output/`이 보이면 준비된 겁니다.

### 2단계. Claude 실행

```bash
cd ~/Desktop/Workspace_Builder
claude
```

Claude Code가 열리면 아래처럼 시작합니다.

```text
워크스페이스 만들어줘
```

### 3단계. 실습 기록 폴더 준비

생성되는 에이전트는 `~/Desktop/Workspace_Builder/90-output/` 아래에 생깁니다. 강의 실습 기록은 따로 남겨야 하니, `bptc-cc` 안에도 기록 폴더를 만듭니다.

```text
~/Desktop/bptc-cc/50-my-work/Part05-에이전트구축/실습25-내에이전트/ 폴더를 만들고, 이번에 만들 에이전트 이름, 선택한 아키타입, 생성 위치, 인수인계 내용을 README.md로 정리할 준비를 해 주세요.
```

### 셋업 완료 체크

준비가 끝나면 두 위치가 있어야 합니다.

- `~/Desktop/Workspace_Builder/` — Workspace Builder를 실행하는 곳
- `~/Desktop/bptc-cc/50-my-work/Part05-에이전트구축/실습25-내에이전트/` — 강의 기록을 남기는 곳

## 🚶 진행 흐름

### 1. Phase 0 — 새 워크스페이스라고 알려주기

처음 질문에서 기존 프로젝트 개선인지, 새로 만드는 것인지 확인할 수 있습니다. 오늘은 새로 만드는 실습입니다.

```text
새 워크스페이스를 만들고 싶습니다. 목적은 [내 업무]를 반복해서 도와주는 전용 에이전트입니다. 기존 워크스페이스 개선이 아니라 신규 생성으로 진행해 주세요.
```

예를 들면 이렇게요.

```text
새 워크스페이스를 만들고 싶습니다. 목적은 BPTC 자재관리 업무에서 입출고 자료를 읽고 재고 차이와 확인할 항목을 정리해주는 전용 에이전트입니다. 기존 워크스페이스 개선이 아니라 신규 생성으로 진행해 주세요.
```

### 2. Phase 1 — 컨텍스트를 충분히 주기

여기서 대충 말하면 뒤가 다 흔들립니다. 누가 쓰는지, 어떤 파일을 넣는지, 결과물이 무엇인지 말해주세요.

```text
이 에이전트의 사용자는 [사용자]입니다. 주로 받는 입력은 [입력 자료]이고, 만들어야 하는 결과물은 [결과물]입니다. 현재 가장 귀찮은 반복 작업은 [반복 작업]입니다.
```

좋은 예시는 이렇습니다.

```text
사용자는 물류 운영 담당자입니다. 주로 받는 입력은 주간 입출고 엑셀, 재고 현황표, 현장 메모입니다. 만들어야 하는 결과물은 확인이 필요한 품목 목록, KPI 요약, 다음 조치 체크리스트입니다. 가장 귀찮은 반복 작업은 숫자를 보고 이상한 품목을 골라낸 뒤 회의용 문장으로 바꾸는 일입니다.
```

### 3. Phase 2 — 아키타입 추천 확인하기

Workspace Builder가 아키타입을 추천하면 Clip 5에서 만든 `my-archetype.md`와 비교합니다. 추천이 다르다고 틀린 건 아닙니다. 이유가 더 납득되는 쪽을 고르면 돼요.

```text
Clip 5에서 저는 메인 아키타입을 [내가 고른 아키타입]으로 정리했습니다. 지금 추천한 아키타입과 비교해서 어떤 쪽이 첫 버전에 더 적합한지 설명해 주세요. 최종 선택은 제가 확인한 뒤 진행해 주세요.
```

여기서 "네, 그걸로 해주세요"라고 바로 넘어가도 되지만, 저는 한 번 비교해보는 걸 추천합니다. 이 과정에서 에이전트의 성격이 많이 선명해져요.

### 4. Phase 3 — 커맨드와 워크플로우 설계하기

이제 실제로 에이전트를 어떻게 부를지 정합니다. 커맨드는 많을수록 좋은 게 아닙니다. 처음에는 4~5개면 충분해요.

```text
첫 버전 커맨드는 4~5개로 제한하고 싶습니다. 각 커맨드에 대해 1) 언제 쓰는지, 2) 입력으로 무엇을 받는지, 3) 어떤 파일 또는 요약을 만드는지 정리해 주세요. 너무 큰 커맨드는 나눠 주세요.
```

예를 들어 Research형 에이전트라면 `/research`, `/verify`, `/synthesize`, `/report`가 자연스럽습니다. Development형이면 `/specify`, `/design`, `/implement`, `/test`가 더 맞겠죠.

### 5. Phase 4 — 생성 전 최종 검토하기

설계가 좋아 보여도 바로 생성하지 말고 한 번 멈춥니다. 빠진 입력, 너무 큰 범위, 위험한 파일 접근이 없는지 봐야 합니다.

```text
생성 전에 최종 설계를 검토해 주세요. 특히 1) 첫 버전 범위가 너무 큰지, 2) 사용자가 매번 넣어야 할 입력이 분명한지, 3) 결과물이 파일로 남는지, 4) 개인정보나 회사 민감정보를 다루는 부분이 있는지 확인해 주세요.
```

다음 Clip 7에서는 이 Phase 4 설계를 Agent Council로 한 번 더 검토합니다. 오늘 시간이 빠듯하면 Claude 단독 검토까지만 하고, Clip 7에서 Council을 소집해도 됩니다.

### 6. Phase 5~6 — 생성하고 인수받기

최종 확인이 끝나면 생성합니다.

```text
검토 내용을 반영해서 워크스페이스를 생성해 주세요. 생성 후에는 위치, 시작 방법, 만들어진 핵심 파일, 첫 실행 예시를 인수인계 형식으로 정리해 주세요.
```

생성 후에는 보통 이런 위치를 받게 됩니다.

```text
~/Desktop/Workspace_Builder/90-output/{내에이전트}/
```

그다음 새 터미널이나 Claude Code에서 생성된 폴더로 이동합니다.

```bash
cd ~/Desktop/Workspace_Builder/90-output/{내에이전트}
claude
```

### 7. 실습 README로 남기기

마지막은 강의 기록입니다. 생성된 워크스페이스 자체는 `90-output`에 있고, 실습 기록은 `bptc-cc`에 남깁니다.

```text
이번 실습 내용을 ~/Desktop/bptc-cc/50-my-work/Part05-에이전트구축/실습25-내에이전트/README.md로 정리해 주세요. 포함할 내용은 에이전트 이름, 생성 위치, 선택한 아키타입, 핵심 커맨드, 첫 실행 방법, 다음 Clip 7에서 Council로 검토할 질문입니다.
```

## 📦 결과물

이번 클립의 실제 결과물은 두 곳에 나뉩니다. 하나는 생성된 에이전트 워크스페이스, 하나는 강의 실습 기록입니다.

```text
~/Desktop/Workspace_Builder/90-output/{내에이전트}/
├── CLAUDE.md
├── README.md
├── .claude/
│   └── commands/
└── ...
```

```text
50-my-work/Part05-에이전트구축/실습25-내에이전트/
└── README.md
```

`README.md`에는 다음 항목을 남깁니다.

- 만든 에이전트 이름: 예를 들어 `BPTC 자재관리 에이전트`
- 생성 위치: `~/Desktop/Workspace_Builder/90-output/{내에이전트}/`
- 선택한 아키타입과 이유
- 핵심 커맨드 4~5개
- 첫 실행 방법: `cd ... && claude`
- Clip 7에서 Council로 검토할 질문

이제 여러분은 그냥 프롬프트 모음이 아니라, 폴더와 설정을 가진 전용 에이전트를 하나 갖게 됩니다.

## 🆘 자주 막히는 포인트

막히는 데는 보통 정해져 있어요. 특히 경로와 역할 혼동이 많이 나옵니다.

| 증상 | 원인 | 해결 |
|---|---|---|
| `~/Desktop/Workspace_Builder`가 없음 | 자료 폴더를 Desktop에 안 둠 | 강의 자료의 `Workspace_Builder` 폴더를 Desktop으로 옮기면 돼요 |
| `claude`가 안 열림 | Claude Code CLI 설치나 로그인 문제 | Part 2 설치 상태를 다시 확인합니다 |
| Builder가 실제 업무를 하려고 함 | "워크스페이스 생성"이라고 말하지 않음 | "신규 워크스페이스 생성"으로 다시 시작하면 돼요 |
| 질문이 너무 많아짐 | 업무 설명이 넓음 | 입력 자료, 결과물, 첫 버전 범위를 다시 좁히세요 |
| 아키타입 추천이 Clip 5와 다름 | 설명한 업무가 다르게 들림 | 두 선택지를 비교해달라고 하면 됩니다 |
| 생성 위치를 못 찾음 | `90-output` 확인을 안 함 | `ls ~/Desktop/Workspace_Builder/90-output`을 보면 돼요 |
| 새 에이전트가 바로 안 잡힘 | 생성된 폴더로 이동하지 않음 | `cd 90-output/{내에이전트} && claude`로 다시 실행하세요 |

여전히 헷갈리면 "지금 저는 Workspace Builder 안에 있나요, 생성된 에이전트 안에 있나요?"라고 물어보세요. 이 질문 하나면 길이 다시 보입니다.
